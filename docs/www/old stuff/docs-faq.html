Frequently asked questions

What is the goal of onesocialweb?

The purpose of onesocialweb is to enable free, open, and decentralized social applications on the web. Its protocol can be used to turn any XMPP server into a full fledged social network, participating in the onesocialweb federation. The suite of extensions covers all the usual social networking use cases such as user profiles, relationships, activity streams and third party applications. In addition, it provides support for fine grained access control, realtime notification and collaboration.

Who owns onesocialweb?

Just like nobody owns the web, email or IM, we think no entity should own or be in full control of onesocialweb. Depending on how onesocialweb evolves it could be incorporated in existing standards bodies like the XMPP foundation or maybe a separate foundation is justified. The onesocialweb code will be open sourced with the liberal Apache 2 license so that everyone can use it as they see fit.

Why an XMPP based architecture?

There are many reasons. There are several similarities between social networks and XMPP based systems: a unique id, a list of friends and relationships. With the basics in place, this primarily needs to be extended and complemented with activities. On top it offers a great and secure authentication framework as well as several interesting add ons out of the box like real-time communication and chatting. Finally, there are several excellent open source projects that can be relatively easily extended using a plugin mechanism. This means we can turn existing XMPP servers e.g. used for instant messaging into a full fledged social web node.
