Developers overview

The OneSocialWeb project team is not currently driving active development of the source code, but we continue our work on specifications and standards to make the dream of an open social web a reality.



State of the social web

Interview by Robert Scoble

Social networks are an increasingly important part of our daily communication. Yet they are closed environments that do not allow you to friend, message or share across networks.

We dream of a world where all social networks are connected and work together in One Social Web. The OneSocialWeb project aimed at defining the language to bridge these networks and make it easy for social networks to join.

What makes this approach unique is that XMPP was used as the engine. Please check the Why use XMPP? section for an explanation on why we picked this.

If you are excited about this approach, you can do several things:

  • For client developers: get the source code and play with our Java libraries and clients for Web and Android. Not into Java? Create an implementation of your choice using the OneSocialWeb protocol.
  • For server developers: get the source code and play with our OpenFire plugin. Not into Java or OpenFire? Create an implementation or plugin for your server of choice using the OneSocialWeb protocol or develop an external XMPP component.
  • Download the OpenFire plugin, setup your own server and join the federation.
  • Share your thoughts on the mailinglist.
